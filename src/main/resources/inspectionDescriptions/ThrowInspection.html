TornadoVM does not support Traps/Exceptions.

On GPUs there is little support for exceptions. For example, on a
division by 0 scenario, the CPU sets a flag in one of the special
registers. Then the Operating System can query those special
registers and pass that value to the application runtime (in this
case, the Java runtime). Then the Java runtime handles the exception.

However, there is no such mechanism on GPUs (link)
which means that TornadoVM must insert extra control-flow to
guarantee that those exceptions never happen. Currently, since
TornadoVM compiles at runtime, many of those checks can be
assured at runtime.
